Pajek Learning

Jul 16, 2019

Pajek is a visualize application. It usually used to show the relationship between target according data.

Data Structure

Network

  • Vertex / node

    Data point

  • Arc / edge

    Line which connect vertex/node and show relation between them

Partition

Sort of vertex/node

Vector

a list of nodes with values

Ex: Degree of nodes

Export to other application

Pajek provide tools help us to export the result to other application or programming, such as Excel, R, SPSS.

Usage

Tools -> (Choose the application)

Centrality

How node important is in the network

Degree/weighted degree centrality

Degree centrality represents how many nodes the target node connect.

Meaning: influence of target node.

Calculate

degree = in-degree + out-degree = 2 + 4 = 6

weighted degree = weighted in-degree + weighted out-degree

  • In-degree

    In patent citation network in-degree means backward citation

  • Out-degree

    In patent citation network in-degree means forward citation

Meaning in patent citation network

Out-degree and in-degree have different meaning, usually observe respectively.

  • Out-degree is high => patent is influential in the development of follow-up technology
  • In-degree is high => patent

Betweenness centrality

Betweenness centrality represents the importance of target node’s position.

Meaning: Is the target node in the key position?

Calculate

Formula

For each pair of nodes in network

$$
\sum \frac{Number\ of\ shortest\ path\ from\ one\ to\ other\ one\ via\ target\ node}{Number\ of\ shortest\ path\ from\ one\ to\ other\ one}
$$

Example

  1. Target node: 3
  2. 1-4, 2-4, 4-5, 4-6, 4-7: 1 / 1 = 1
  3. 1-5, 1-5, 1-7: 1 / 2 = 0.5
  4. 1*5 + 0.5*3 = 6.5

Closeness centrality

Closeness centrality represents how close the target node with each others is?

Meaning: How the target node close to the center of the network?

Calculate

Formula

$$
\sum\frac{1}{Average\ of\ shortest\ path\ from\ target\ node\ to\ other\ node}
$$

Calculate centrality with Pajek

Create

Pajek calculate value of each node and store in vector

Network -> Create Vector -> Centrality -> (Choose Degree/Closeness/Betweenness)

Sorted

Patent Island (Partition by line weight)

Put similar patent together for further analysis

Ex: Patents have similar IPC/CPC number or keywords

Filter

Kick out irrelevant patents. For instance, patents have no connect with other patents.

Usage

Network -> Create Partition -> Islands -> Line Weights
