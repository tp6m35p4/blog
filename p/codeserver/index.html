在iPadOS上用VS Code吧！

code server 與 iPadOS: 多一個買iPad的理由

最近在找iPad Pro能夠發揮生產力的場景，於是就想到了code server

我用的環境

OS: centOS 7

code server 是什麼

根據他們Github上的敘述，他就是瀏覽器上的VS Code (VS Code in the browser)，其實就是可以透過瀏覽器去遠端到伺服器中的VS Code，更白話就是可以在網頁上寫程式啦。

code server 的安裝

其實安裝很簡單，照著Github上面的步驟去走就可以了

複製貼上就可以了

curl -fsSL https://code-server.dev/install.sh | sh

code server有兩種連線方式，一種是透過SSH forwarding來連線，另一種是把這個服務公開到外網上

更換登入密碼

登入密碼寫在~/.config/code-server/config.yaml這個檔案裡面，去換成自己喜歡的吧～

Nginx設定

下面是範例的conf內容，比較要注意的是因為code server有使用web socket，所以要加上跟web socket有關的設定。

server {
    listen 80;
    listen [::]:80;
    server_name mydomain.com;

    location / {
      proxy_pass http://localhost:8080/;
      proxy_set_header Host $host;
      # for web socket
      proxy_set_header Upgrade $http_upgrade;
      proxy_set_header Connection upgrade;
      proxy_set_header Accept-Encoding gzip;
    }
}

重新啟動code server

sudo systemctl restart code-server@$USER

要用在iPad上必須要做的事

必須要有HTTPS、必須要有HTTPS、必須要有HTTPS，很重要所以說三次

CloudFlare的半套呢？

在使用Let’s Encrypt之前我是用CloudFlare提供的半套HTTPS，也就是說從瀏覽器到CloudFlare之間有保護，從CloudFlare到伺服器是沒保護的，這個做法在windows跟macOS上是沒有問題的，
罷特！當要從iPadOS上使用這個服務的時候，會沒辦法建立web socket的連線，所以還是得乖乖用憑證啦～

後話

在設定完code server後就馬上用他來寫了一個小專案，不得不說把開發環境放在伺服器上還是比較方便，不用怕連線不穩定、還可以有固定IP。另外在iPad上寫程式的體驗說實在的還不錯，不過長時間或開發大型專案可能會不好抄code有點吃力，重點是，我好想買鍵盤RRRRRR。

最近的大新聞就是以後的mac要換成arm架構了，期待會有更多具有生產力的軟體可以在iPad上使用到。
